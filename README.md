# Roseville Golf Booking Bot

## Deploying to Render

### 1. Push to GitHub
Create a new **private** GitHub repo and push all these files to it.

```bash
git init
git add .
git commit -m "Initial deploy"
git remote add origin https://github.com/YOUR_USERNAME/YOUR_REPO.git
git push -u origin main
```

### 2. Create Render service
1. Go to https://render.com and sign in
2. Click **New → Blueprint** and connect your GitHub repo
3. Render will detect `render.yaml` and configure everything automatically
4. Click **Apply** — the first build takes ~5 minutes (installing Chrome)

### 3. Add users
After the first deploy, SSH into your Render service (or use the Shell tab) and create `/data/users.json`:

```json
{
  "4291": { "password": "NewcastleTaree1!", "name": "Ross Farrelly" },
  "1234": { "password": "theirpassword",    "name": "Dan Pryor" }
}
```

The `SECRET_KEY` is auto-generated by Render — you don't need to set it manually.

### 4. Persistent data
The `/data` disk stores:
- `users.json` — who can log in
- `member_preferences_<id>.json` — per-user preferences
- `member_cache.json` — cached member directory

This survives redeploys. The disk is included in the Starter plan ($7/month).

## Running locally

```bash
pip install -r requirements.txt
python3 local_golf_server.py
# Open http://127.0.0.1:5001
```
